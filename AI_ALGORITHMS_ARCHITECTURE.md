# LINKUP - AI Models & Algorithmic Techniques Architecture

This document outlines the core algorithms, artificial intelligence models, and data processing techniques utilized in the LINKUP platform. It is designed to serve as a technical reference for project guides and evaluators.

---

## 1. Hybrid ATS Matchmaking Engine (Deterministic + Semantic)
The Applicant Tracking System (ATS) scoring does not rely on a single technique but utilizes a **Hybrid Scoring Engine** that combines traditional parsing with modern semantic analysis.

*   **Weighted Ensemble Scoring Algorithm**: Computes a deterministic baseline score using a multi-variable linear equation.
    *   `Final ATS Score = (Skills × 0.35) + (Experience × 0.25) + (Projects × 0.20) + (Education × 0.10) + (Certifications × 0.10)`
    *   **Dynamic Contextual Weighting**: The algorithm dynamically adjusts logic based on candidate experience. For example, in *Fresher Mode*, it prioritizes internship keywords and project domains, whereas in *Experienced Mode*, it uses a ratio of `(Candidate Years / Required JD Years)`.
*   **Constant-Time (O(1)) Hash Map Lookups**: Uses highly optimized Set-based lookups (`O(1)` time complexity) against curated vocabularies (`skills_list.json`, `certificates_list.json`) to perform rapid Exact/Partial string matching and alias resolution.

## 2. Generative AI & Natural Language Processing (LLM)
The platform integrates advanced Large Language Models (LLMs) to perform tasks that traditional regex or keyword-matching algorithms fail to achieve.

*   **Local OSS LLMs (Ollama)**: The architecture supports running models locally (e.g., `qwen`, `llama3`, `deepseek`) ensuring 100% data privacy for sensitive resumes.
*   **Structural Parsing via Prompt Engineering**: Uses **Zero-Shot** and **Few-Shot Prompting** techniques to perform entity extraction. It converts unstructured raw PDF text into strongly typed JSON schemas (extracting exact years of experience, parsing project descriptions, etc.).
*   **Deep Semantic JD Matching**: Instead of just looking for exact words, the LLM performs contextual analysis. It evaluates the *depth* of experience in a specific skill and understands semantic relationships (e.g., knowing that "React" implies familiarity with "Frontend Frameworks").
*   **Automated Skill Gap Analysis**: Generates personalized learning paths (Market Insights) by calculating the difference between a candidate's recognized skills and the current market requirements.

## 3. Information Retrieval & Recommendation Engine
The platform uses lightweight, high-performance information retrieval concepts to match candidates with jobs in real-time.

*   **Content-Based Filtering**: The recommendation service (`recommendationService.py`) ranks jobs based on the similarity between the candidate's parsed profile and the Job Tags.
*   **Jaccard Similarity (Set Intersection)**: To compute job recommendations, it calculates the intersection of sets (`User_Skills ∩ Job_Tags`). The base match score is dynamically increased (e.g., `+6%` per matched skill), sorting jobs in descending order to output the `Top-N` highest relevancy roles.

## 4. High-Concurrency AI Queue Management
Handling LLM requests can be a bottleneck. The project utilizes distributed system concepts to handle high traffic seamlessly.

*   **Asynchronous Event Loop & Thread Pooling**: Integrates Python's `asyncio` with a `ThreadPoolExecutor` so that heavy, CPU-bound LLM inferences do not block the FastAPI web server.
*   **Round-Robin API Key Rotation**: Implemented a multi-key fallback mechanism. If an LLM provider hits a rate limit (HTTP 429), the queue algorithm automatically rotates to the next available API key or fallback model, ensuring maximum uptime.
*   **Token Bucket / Queue Logic**: Limits the number of parallel LLM inferences (`OLLAMA_MAX_PARALLEL_REQUESTS`) to prevent memory exhaustion on the server.

## 5. Trust & Verification Algorithms
To ensure the integrity of the platform, several programmatic verification techniques are applied to candidate data.

*   **3-Layer Certificate Forensics**:
    1.  *API Resolution*: Validates credential IDs against external issuers.
    2.  *Digital Forensics (EXIF Analysis)*: Uses the `Pillow` library to extract image metadata, detecting if a certificate image has been digitally manipulated or generated by Photoshop.
    3.  *Anomaly Detection*: Flags suspicious patterns and queues them for admin review.
*   **Web Scraping & HTTP Resolution**: Uses web crawling techniques to ping provided GitHub and Portfolio URLs to ensure the links are active (HTTP 200) before awarding ATS points.

## 6. Unstructured Data Extraction Pipeline
*   **PDF OCR & Layout Analysis**: Utilizes `PyMuPDF` (`fitz`) for fast, low-level textual extraction from complex PDF layouts, preserving the reading order of multi-column resumes.
*   **Heuristic Text Cleaning**: RegEx (Regular Expressions) pipeline is used to sanitize, trim, and normalize unstructured strings prior to LLM injection to reduce token usage and hallucination rates.
